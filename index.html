About the Project

This website is created just to check how backend works. It includes 4 microservices:

IAM-service - Handles user registration and login, creation of users, posts, and comments. It also includes JWT setup, Security, Flyway, CORS, and PostgreSQL.
UTILS-service - Collects action logs, such as user creation, and post and comment creation. It receives all information from IAM-service via Kafka.
GATEWAY-service - Centralizes all microservices through a single Gateway service.
DISCOVERY-service - Uses Consul for dynamic service registration.

Access and Restrictions

This project includes three roles: User, Admin, and Super Admin.

Upon registration, a user is automatically assigned the role of User. This role grants access to basic features like viewing and creating content.

Admin role can be assigned to a user only by a Super Admin.

Super Admin is the highest level of access and holds the authority to assign or revoke the Admin role, as well as oversee all aspects of the system.
